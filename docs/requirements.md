# 머니로그(MoneyLog) 요구사항 정의서

- 작성자: 전기혁
- 작성일: 2026-07-22
- 프로젝트 기간: 5일

## 1. 프로젝트 개요

- 한 줄 소개: 개인이 수입/지출을 기록하고 카테고리별·월별 통계를 보는 가계부 웹 서비스
- 대상 사용자: 로그인한 개인 사용자(내 데이터는 나만 접근)
- 목표: 부트캠프 전 과정 복습 + EC2 실배포까지 완주

## 2. 유저 스토리

| # | 유저 스토리 (~로서 나는 ~하고 싶다, 왜냐하면 ~) |
|---|-----------------------------------------------|
| US-1 | 사용자로서 나는 이메일과 비밀번호로 회원가입/로그인하고 싶다, 왜냐하면 내 가계부를 나만 보고 싶기 때문이다. |
| US-2 | 사용자로서 나는 수입과 지출을 등록하고 싶다, 왜냐하면 돈이 어디서 들어오고 나가는지 기록해야 하기 때문이다. |
| US-3 | 사용자로서 나는 등록한 거래를 수정·삭제하고 싶다, 왜냐하면 금액이나 날짜를 잘못 입력할 수 있기 때문이다. |
| US-4 | 사용자로서 나는 거래를 식비·교통 같은 카테고리로 분류하고 싶다, 왜냐하면 어디에 돈을 많이 쓰는지 알고 싶기 때문이다. |
| US-5 | 사용자로서 나는 이번 달 거래를 카테고리·타입으로 걸러서 보고 싶다, 왜냐하면 원하는 항목만 빠르게 확인하고 싶기 때문이다. |
| US-6 | 사용자로서 나는 이번 달 총수입·총지출·잔액과 카테고리별 지출을 보고 싶다, 왜냐하면 한 달 예산 상태를 한눈에 파악하고 싶기 때문이다. |
| US-7 | 사용자로서 나는 내 데이터에만 접근하고 싶다, 왜냐하면 남이 내 가계부를 보거나 고치면 안 되기 때문이다. |
| US-8 | (도전) 사용자로서 나는 카테고리별 월 예산을 정하고 초과 시 경고를 받고 싶다, 왜냐하면 과소비를 막고 싶기 때문이다. |

## 3. 기능 요구사항 (MoSCoW)

| ID | 기능 | 설명 | 우선순위(Must/Should/Could) |
|----|------|------|:---:|
| F-01 | 회원가입/로그인 | JWT 발급, BCrypt | Must |
| F-02 | 카테고리 관리 | 시드 + CRUD | Must |
| F-03 | 거래내역 CRUD | 수입/지출 등록·조회·수정·삭제 | Must |
| F-04 | 목록 필터·페이징 | yearMonth/type/categoryId + page/size | Must |
| F-05 | 월별 통계 | 총수입·지출·잔액, 카테고리별 지출 | Must |
| F-06 | 인가 | 본인 데이터만 | Must |
| F-07 | 검증·예외 | 금액>0, 날짜 필수, {success:false, code, message, data:null} | Must |
| F-08 | 프론트 4종 | 로그인/목록/등록/통계 | Should |
| F-09 | Swagger | API 문서화 | Should |
| F-10 | 배포 | Docker+GitHub Actions+EC2 | Should |
| F-11 | 월 예산·초과 경고 | 카테고리/전체 월 예산(Budget) 설정, 초과 시 경고 | Could (도전) |
| F-12 | 통계 시각화 | 카테고리 파이차트, 월별 추이 라인차트 | Could (도전) |
| F-13 | 검색·CSV 내보내기 | 키워드/기간/금액범위 검색, CSV(엑셀) 내보내기 | Could (도전) |
| F-14 | Refresh Token·소셜 로그인 | Refresh Token 재발급 흐름, (선택) 소셜 로그인 | Could (도전) |
| F-15 | 테스트·커버리지 | 단위/통합 테스트 + 커버리지 측정 | Could (도전) |
| F-16 | Jenkins·K8s·무중단 배포 | Jenkins 파이프라인, K8s(Deployment/Service/Ingress), 롤링 업데이트 | Could (도전) |

## 4. 비기능 요구사항

| 분류 | 요구사항 | 구체적 기준 |
|------|----------|-------------|
| 🔐 보안 | 본인 데이터만 | user_id 기준 필터 |
| 🔐 보안 | 비밀번호 보호 | 비밀번호는 평문 저장 금지, BCrypt 해시로 저장 |
| 🔐 보안 | 인증 방식 | 인증 필요 API는 `Authorization: Bearer {accessToken}` 헤더의 JWT로 검증 |
| ⚡ 성능 | 목록 페이징 | 거래내역 목록은 page/size 페이징으로 조회 (전체 로딩 금지) |
| ⚡ 성능 | 통계 응답 | 월별 통계는 DB 집계 쿼리로 계산해 빠르게 응답 |
| ✅ 검증 | 입력 유효성 | amount > 0, transactionDate 필수, type은 INCOME/EXPENSE만 허용 |
| ✅ 품질 | 일관된 에러 응답 | 전역 예외 처리로 모든 에러를 `{success:false, code, message, data:null}` 형태로 통일 |
| 🚀 배포 | 실제 배포·접속 | AWS EC2에 배포하여 실제 접속 가능한 URL 제공 |
| 📖 문서 | API 문서화 | Swagger UI로 API 명세 확인 가능 |

## 5. 범위(Scope)

| 구분 | 기본 과제 범위 (In Scope, Must/Should) | 도전 과제 범위 (Stretch, Could) | 이번엔 안 함 (Out of Scope, Won't) |
|------|------|------|------|
| 인증 | 이메일 회원가입/로그인, JWT 발급 | Refresh Token 재발급, 소셜 로그인 | 이메일 인증, 비밀번호 찾기 |
| 거래 | 수입/지출 CRUD, 월·카테고리·타입 필터, 페이징 | 키워드/기간/금액 검색, CSV 내보내기 | 반복 거래, 첨부 영수증 이미지 |
| 통계 | 월별 총수입·지출·잔액, 카테고리별 지출 | 파이/라인 차트 시각화 | 연간 리포트, 카테고리 예측 |
| 예산 | (없음) | 월 예산 설정 + 초과 경고 | 알림 이메일/푸시 발송 |
| 프론트 | 화면 4종(로그인/목록/등록/통계) | 반응형·차트 UI | 모바일 앱, 다크모드 |
| 배포 | Docker + GitHub Actions + EC2 | Jenkins, K8s, 무중단 배포 | 오토스케일링, 멀티리전 |
| 품질 | 입력 검증, 전역 예외, Swagger | 단위/통합 테스트 + 커버리지 | 성능 부하 테스트 |
